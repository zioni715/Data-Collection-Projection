## 4) 업무 흐름(전환 패턴) 상위
가장 자주 반복되는 전환 흐름:
- CHROME → CODE (브라우저에서 정보 확인 후 코드 작성)
- CODE → CHROME (코드 작성 중 참고/검색 전환)
- CODE ↔ KAKAOTALK (작업 중 커뮤니케이션 반복)
- CODE ↔ NOTION (정리/노트 작성 병행)
- CODE ↔ WINDOWSTERMINAL (빌드/실행/명령 작업 반복)

**해석**: “검색/참고 → 코드 작성 → 커뮤니케이션/정리 → 다시 코드” 사이클.

## 5) 1~5차 수집별 특징 요약
- **1차**: CODE 중심, 브라우저/노션/카톡이 보조
- **2차**: 카톡 비중 증가, 코드와 브라우저 균형
- **3차**: 터미널/코드 비중 상승 (개발 루틴 강화)
- **4차**: 앱 다양성↑, 전환 패턴이 풍부
- **5차**: 카톡/브라우저 비중↑, 세션 수는 많지만 요약/패턴은 비어있음(요약 생성 누락)

---

## 6) “이 사람은 9시에 뭐하고 10시에 뭐한다” 해석 예시
- **09~11시**: 집중 개발(코딩 + 터미널 + 참고 검색)
- **12시**: 메신저 중심(카톡/디스코드)
- **13~15시**: 다시 개발 모드 복귀
- **16~17시**: 커뮤니케이션 비중 증가

## 7) 패턴 품질을 더 높이기 위한 제안
1. **duration_sec 합산 분석** 추가 → 건수 대신 실제 사용 시간 기반 패턴 도출
2. **요일별 패턴** 분리 (월/화/수/목/금 차이 확인)
3. **Focus block 길이 통계** 추가 (짧은 전환 노이즈 제거)
4. **메신저/브라우저 상세**는 마스킹 상태 유지하되 “도메인/타이틀 힌트”만 요약에 반영

## 8) 한 줄 요약 (초개인화 관점)
이 사용자는 **오전~오후 초반은 개발 집중(CODE/Terminal/Chrome)**, **오후 후반은 커뮤니케이션 비중 증가(KakaoTalk/Discord)** 패턴을 보이며, **브라우저 ↔ 코드 ↔ 노션** 전환이 가장 큰 반복 루틴이다.
